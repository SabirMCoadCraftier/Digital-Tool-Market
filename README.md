## 🚀 Project Overview

A fully responsive **digital tools marketplace** where users can explore products, manage their cart, and enjoy a smooth interactive experience — all built with modern frontend technologies.

---

## ✨ Key Features

| Feature | Description |
|---|---|
| 🗂️ Dynamic Product Listing | 6–10 tools loaded from JSON data |
| 🛒 Interactive Cart System | Add, Remove & Clear cart |
| 🔁 View Toggling | Seamless switch between Products & Cart |
| 🔔 Toast Notifications | Real-time alerts using React-Toastify |
| 📱 Fully Responsive | Works on all screen sizes |
| 🎨 Clean UI | Stats, Pricing & Steps based on Figma design |

## 🧠 Challenges I Solved

> One of the biggest challenges was **managing cart state efficiently** — ensuring real-time updates, accurate product counts, and smooth user interaction. Implementing clean state management and handling UI updates properly was a great learning experience.

This project strengthened my:

## 🚀 Getting Started

```bash
# Clone the repository
git clone https://github.com/SabirMCoadCraftier/Digital-Tool-Market.git

# Navigate to project folder
cd Digital-Tool-Market

# Install dependencies
npm install

# Start development server
npm run dev
```

# React + Vite

This template provides a minimal setup to get React working in Vite with HMR and some ESLint rules.

Currently, two official plugins are available:

- [@vitejs/plugin-react](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react) uses [Oxc](https://oxc.rs)
- [@vitejs/plugin-react-swc](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react-swc) uses [SWC](https://swc.rs/)

## React Compiler

The React Compiler is not enabled on this template because of its impact on dev & build performances. To add it, see [this documentation](https://react.dev/learn/react-compiler/installation).

## Expanding the ESLint configuration

If you are developing a production application, we recommend using TypeScript with type-aware lint rules enabled.
